# 软件设计说明书

## 软件名称：Python程序设计在线编程评测与自动评分系统

---

## 一、概述

### 1.1 开发背景

在高校Python程序设计课程教学中，传统的人工批改代码作业方式存在以下问题：
- 教师批改工作量大，反馈周期长
- 学生无法即时获得代码运行结果的反馈
- 缺乏统一的代码质量评价标准
- 抄袭检测困难

为解决上述问题，开发本系统，提供在线编程、自动评测、成绩统计、代码查重等一站式服务。

### 1.2 软件功能简介

本系统是一个基于B/S架构的在线编程评测平台，具备以下主要功能：

1. **题库管理**：教师可创建、编辑、发布编程题目，支持Markdown格式描述
2. **在线编程**：学生在线编写Python代码，支持语法高亮
3. **自动评分**：系统在安全沙箱中执行学生代码，逐测试用例比对输出，自动评分
4. **测试用例管理**：教师为每道题配置多个测试用例，支持公开/隐藏用例
5. **提交记录**：完整记录每次提交的代码、得分、执行结果
6. **班级管理**：教师创建班级，学生通过邀请码加入
7. **成绩统计**：按题目统计通过率、平均分、最高分等
8. **代码查重**：自动检测学生提交代码的相似度
9. **考试/作业模式**：支持限时考试、作业截止等

### 1.3 运行环境

| 项目 | 规格 |
|------|------|

## 四、核心功能设计

### 4.1 自动评分引擎流程

自动评分是本系统的核心功能，执行流程如下：

```
学生提交代码
    ↓
┌───────────────────────────────────────────────┐
│                  安全校验                       │
│  (1) 检查代码长度                             │
│  (2) 检查危险模块（os, sys, socket等）        │
│  (3) 检查危险函数（eval, exec等）             │
└───────────────────────────────────────────────┘
    ↓ 通过
┌───────────────────────────────────────────────┐
│              逐测试用例执行                     │
│                                               │
│  for each 测试用例:                           │
│  ├─ 构造 stdin 输入                           │
│  ├─ subprocess执行Python代码                  │
│  ├─ 捕获 stdout/stderr                       │
│  ├─ 计时（超时控制）                          │
│  ├─ 输出规范化比对（去空白、统一换行符）       │
│  └─ 记录该用例通过/失败及得分                 │
└───────────────────────────────────────────────┘
    ↓
┌───────────────────────────────────────────────┐
│              汇总评分结果                       │
│  (1) 所有用例通过 → accepted                  │
│  (2) 部分用例失败 → wrong                     │
│  (3) 代码有错误 → error                       │
│  (4) 记录总得分、总执行时间                    │
└───────────────────────────────────────────────┘
    ↓
┌───────────────────────────────────────────────┐
│              结果写入数据库                     │
│  (1) 更新 submission 记录                     │
│  (2) 逐条写入 test_results                    │
│  (3) 更新 problem 统计（提交数/通过数）        │
└───────────────────────────────────────────────┘
```

### 4.2 安全沙箱设计

为保证服务器安全，代码执行采用多层防护：

1. **静态代码扫描**：提交时扫描代码，拦截危险模块和函数
2. **资源限制**：限制CPU时间（默认5秒/用例）和内存使用
3. **环境隔离**：在临时目录执行，清空环境变量
4. **自动清理**：执行完成后立即删除临时文件
5. **禁止模块列表**：os, sys, subprocess, shutil, socket, ctypes, signal 等

### 4.3 代码查重算法

采用Token序列分析算法：

```
1. 代码预处理：移除注释、字符串字面量
2. Token化：提取代码中的关键字和标识符
3. 规范化：变量名→VAR，数字→NUM
4. 相似度计算：
   - Jaccard相似度（基于Token集合）
   - N-gram序列相似度（基于Token序列）
   - 综合得分 = Jaccard×0.4 + 序列相似度×0.6
5. 超过阈值（默认75%）标记为可疑
```

### 5.2 界面截图描述

（本系统采用Bootstrap 5响应式UI，支持桌面端和移动端访问）

**管理面板**：统计卡片（用户数、题目数、提交数、通过数）、最近提交列表。

## 七、总结

本系统为高校Python程序设计课程提供了一个完整的在线编程评测解决方案，覆盖了从题目管理、在线编程、自动评分到成绩统计分析的全流程。系统的核心自动评分引擎在安全沙箱中执行学生代码，通过逐测试用例比对输出实现精确评分。代码查重功能帮助教师有效发现抄袭行为。

系统采用Python Flask框架开发，与Python课程教学内容高度契合，便于教师在课堂上进行示范教学，同时也方便学生理解和参与二次开发。
